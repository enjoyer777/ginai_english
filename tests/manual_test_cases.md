# Чек-лист ручного тестирования

Прогон всех функциональных и эксплуатационных сценариев бота. Отмечай галочки `[x]` по мере прохождения. По умолчанию проходи **сверху вниз** — некоторые сценарии полагаются на то, что предыдущие отработали (например, нельзя проверить hot lead, если бот вообще не отвечает).

**Псевдонимы для команд** (вставь свой VPS-host):

```bash
VPS=onedash2-enjoyer777
PRJ=/home/enjoyer777/ginai_english
TG_ID=128657277       # подставь свой Telegram user_id (узнай через A1 ниже)
```

**Шорткаты, которыми ты будешь часто пользоваться:**

```bash
# логи бота
ssh $VPS "cd $PRJ && docker compose logs --tail=40 bot"

# хвост логов в реальном времени
ssh $VPS "cd $PRJ && docker compose logs -f bot"

# рестарт бота (без пересборки) — подхватить .env / прочитать xlsx заново
ssh $VPS "cd $PRJ && docker compose restart bot"

# полный рестарт + пересборка образа (после правок кода без CI)
ssh $VPS "cd $PRJ && docker compose up -d --build"

# список юзеров в локальной БД
ssh $VPS "cd $PRJ && docker compose exec -T bot python -m app.scripts.reset_memory"

# вычистить твою память для следующего сценария
ssh $VPS "cd $PRJ && docker compose exec -T bot python -m app.scripts.reset_memory --user $TG_ID"

# поднять/опустить xray-сайдкар (для негативных тестов)
ssh $VPS "cd $PRJ && docker compose stop xray"
ssh $VPS "cd $PRJ && docker compose start xray"
```

## Что закрывается юнит-тестами (запускается отдельно)
